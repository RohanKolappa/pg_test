CRC-32 1+ [BF] - - - - binary The CRC is computed on all the data of the Master element it's in, regardless of its position. It's recommended to put the CRC value at the beggining of the Master element for easier reading. All level 1 elements should include a CRC-32. Unused.
Void 1+ [EC] - - - - binary Used to void damaged data, to avoid unexpected behaviors when using damaged data. The content is discarded. Also used to reserve space in a sub-element for later use. Used as needed.

Cluster
Cluster 1 [1F][43][B6][75] - * - - sub-elements The lower level element containing the (monolithic) Block structure. Present
Timecode 2 [E7] * - - - u-integer Absolute timecode of the cluster (based on TimecodeScale). Standard. Our timecodes will be offsets (in ms) from the beginning of media subsuming all the gaps. The timecode will thus represent the play-out time of the first frame in the current cluster.
SilentTracks 2 [58][54] - - - - sub-elements The list of tracks that are not used in that part of the stream. It is useful when using overlay tracks on seeking. Then you should decide what track to use. Unused.
SilentTrackNumber 3 [58][D7] - * - - u-integer One of the track number that are not used from now on in the stream. It could change later if not specified as silent in a further Cluster. Unused.
Position 2 [A7] - - - - u-integer Position of the Cluster in the segment (0 in live broadcast streams). It might help to resynchronise offset on damaged streams. Unused.
PrevSize 2 [AB] - - - - u-integer Size of the previous Cluster, in octets. Can be useful for backward playing. Unused.
BlockGroup 2 [A0] - * - - sub-elements Basic container of information containing a single Block or BlockVirtual, and information specific to that Block/VirtualBlock. Unused (we will be using SimpleBlocks only)
Block 3 [A1] * - - - binary (see Block Structure) Block containing the actual data to be rendered and a timecode relative to the Cluster Timecode. N/A
BlockVirtual 3 [A2] - * - - binary (see Block Virtual) A Block with no data. It must be stored in the stream at the place the real Block should be in display order. N/A
BlockAdditions 3 [75][A1] - - - - sub-elements Contain additional blocks to complete the main one. An EBML parser that has no knowledge of the Block structure could still see and use/skip these data. N/A
BlockMore 4 [A6] * * - - sub-elements Contain the BlockAdditional and some parameters. N/A
BlockAddID 5 [EE] * - > 0 1 u-integer An ID to identify the BlockAdditional level. N/A
BlockAdditional 5 [A5] * - - - binary Interpreted by the codec as it wishes (using the BlockAddID). N/A
BlockDuration 3 [9B] - - - TrackDuration u-integer The duration of the Block (based on TimecodeScale). This element is mandatory when DefaultDuration is set for the track. When not written and with no DefaultDuration, the value is assumed to be the difference between the timecode of this Block and the timecode of the next Block in "display" order (not coding order). This element can be useful at the end of a Track (as there is not other Block available), or when there is a break in a track like for subtitle tracks. N/A
ReferencePriority 3 [FA] * - - 0 u-integer This frame is referenced and has the specified cache priority. In cache only a frame of the same or higher priority can replace this frame. A value of 0 means the frame is not referenced. N/A
ReferenceBlock 3 [FB] - * - - s-integer Timecode of another frame used as a reference (ie: B or P frame). The timecode is relative to the block it's attached to. N/A
ReferenceVirtual 3 [FD] - - - - s-integer Relative position of the data that should be in position of the virtual block. N/A
CodecState 3 [A4] - - - - binary The new codec state to use. Data interpretation is private to the codec. This information should always be referenced by a seek entry. N/A
Slices3[8E]-*--sub-elementsContains slices description. N/A
TimeSlice4[E8]-*--sub-elementsContains extra time information about the data contained in the Block. While there are a few files in the wild with this element, it is no longer in use and has been deprecated. Being able to interpret this element is not required for playback. N/A
LaceNumber5[CC]---0u-integerThe reverse number of the frame in the lace (0 is the last frame, 1 is the next to last, etc). While there are a few files in the wild with this element, it is no longer in use and has been deprecated. Being able to interpret this element is not required for playback. N/A
FrameNumber5[CD]---0u-integerThe number of the frame to generate from this lace with this delay (allow you to generate many frames from the same Block/Frame). N/A
BlockAdditionID5[CB]---0u-integerThe ID of the BlockAdditional element (0 is the main Block). N/A
Delay5[CE]---0u-integerThe (scaled) delay to apply to the element. N/A
Duration5[CF]---0u-integerThe (scaled) duration to apply to the element. N/A
SimpleBlock 2 [A3] - * - - binary (see SimpleBlock Structure) Similar to Block but without all the extra information, mostly used to reduced overhead when no extra feature is needed. This is where our media frames will be stored.
EncryptedBlock 2 [AF] - * - - binary (see EncryptedBlock Structure) Similar to SimpleBlock but the data inside the Block are Transformed (encrypt and/or signed). Unused.
